# feat: Upgrade OpenCode to 1.17.3 (action + gateway)

## Overview

Raise the pinned OpenCode/SDK version from 1.15.13 to 1.17.3 across the action/runtime tier and the gateway workspace image, lifting the Renovate cap accordingly. This is a deliberate, reviewed bump past the validated-release cap (OpenCode ships breaking changes in patch releases, so the cap exists precisely to force this review).

The harness package (`packages/harness/**`) has its own base-version + release pipeline and is **out of scope** here — its `1.15.13` references stay untouched.

## Why now

Upstream research (1.15.13 → 1.17.3) found no breaking change for our consumer surfaces: `/event` SSE delivery got *fixes* (1.16.0 subscription-race fix; project-scoped bus delivery fixes), no SSE envelope/route rename, no permission route renames, no config-schema key renames, SDK API stable, bun still `1.3.14`. The two historically risky areas (streaming SyncEvent delivery and directory-scoped SSE) were repaired upstream, not regressed — but they remain the must-smoke-test surfaces because of the 1.14.42-class regression history.

## Goals

- Action/runtime and gateway workspace image both run OpenCode 1.17.3, no deliberate skew.
- Renovate cap raised to `<=1.17.3` for `@opencode-ai/sdk` + `anomalyco/opencode`; bun cap unchanged (still `1.3.14`).
- The streaming contract (`message.part.updated` tool lifecycle + `message.part.delta` text) and directory-scoped SSE are empirically re-validated against a live 1.17.3 server before merge — not assumed.
- Pin-tracking commentary (constants.ts, renovate.json5, Dockerfile header) updated to reflect 1.17.3 as the validated release.

## Non-Goals

- Harness base-version bump (separate pipeline).
- Removing the legacy `session.next.*` fallback handlers in `streaming.ts`/`run-core.ts` — they stay inert-but-retained (cheap insurance against a future re-regression).
- Bun version change.
- Any behavioral refactor of streaming/permission code beyond what 1.17.3 forces (research says: nothing).

## Key Technical Decisions

- **Single PR, action + gateway together.** No deliberate skew this time: research shows no streaming-contract change, so the gateway-renders-tool-progress concern that forced action-first/gateway-follows in the 1.15.13 cutover does not apply. Both move to 1.17.3 in one PR. (If the live probe surfaces any tool-event shape drift, fall back to action-first — see Risks.)
- **Live integration probe is a merge gate, not optional — RECREATE it (it does not exist).** The 1.15.13-era probe was a temporary artifact and was deleted after that bump (verified: no env-gated/live probe exists in `src/features/agent/` or `packages/runtime/src/agent/` on current main). Unit 2 writes a fresh temporary, env-gated probe (run from an isolated 1.17.3 server: temp HOME/XDG, `--pure`, `opencode/big-pickle`, directory-scoped subscribe) that drives the real harness path (`sendPromptToSession → processEventStream → arming → wait`) and asserts a `| Bash` tool line AND final assistant text render over directory-scoped SSE. Local-only, not committed to CI (avoids a live-LLM dependency); evidence recorded in the PR body, then the probe file is removed before the PR (it is scaffolding, not a shipped test).
- **Lockfile + dist.** Bump both `@opencode-ai/sdk` deps, run `pnpm install` (refreshes `pnpm-lock.yaml` and the tsdown license collector), rebuild `dist/`. dist is committed and gated.
- **Workspace image validated by CI smoke.** The existing `workspace-smoke` / Workspace Image Smoke Test downloads the musl tarball for `ARG OPENCODE_VERSION` and boots. The Dockerfile (`deploy/workspace.Dockerfile:55-60`) targets `opencode-linux-x64-baseline-musl.tar.gz` (amd64) / `opencode-linux-arm64-musl.tar.gz` (arm64); the ACTION installer (`src/services/setup/opencode.ts:47`) targets `opencode-linux-x64.tar.gz` / `opencode-darwin-arm64.zip`. **Verified 2026-06-10 via `gh api releases/tags`: v1.15.13 and v1.17.3 ship the IDENTICAL asset set, including all of the above names** — so neither download path's name construction breaks at 1.17.3 (this disproves a review inference that the action path would 404). The smoke job remains the CI authority that the 1.17.3 musl binary downloads + boots.
- **Comment refresh, not removal.** The regression commentary in constants.ts + renovate.json5 is updated to name 1.17.3 as the newly validated release and keep the "deliberate bump past cap" rationale; it is not deleted (the rationale is still true).
